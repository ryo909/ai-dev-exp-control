# learned_rules

このファイルは週次バッチで学習した運用知見を追記するためのプレースホルダです。
- 契約優先順位: `system/contract.md` を最優先
- 追記専用で運用し、既存ルールの破壊的置換は行わない
- 直近14本と同一familyの企画は原則不採用（同バッチ内は最大1本）。
- 同一バッチ内で theme / mechanic / input_style / audience_promise の連発を避ける。
- titleやone_sentenceが既存公開分と近似する場合は、採用前に別案へ差し替える。
- 企画選定時は除外理由を `plans/candidates/dayNNN_novelty_selection.json` に残す。
