# RULES.md — AI契約ルール

このファイルは、AI（Antigravity / Codex等）がパイプライン実行時に **必ず遵守する** ルールを定義する。

---

## 1. 破壊的変更の禁止

- 既存ファイル・既存repoの **削除・上書き・force push・大量改変** は禁止。
- 破壊的変更が必要な場合は **必ず停止** し、以下を提示して人間の承認を待つ：
  - 理由
  - 影響範囲
  - 代替案

## 2. 再開可能性が最優先

- 電源断・中断を前提に設計する。
- **STATE.json だけを真実** として、いつでも続行できる状態を維持する。
- 毎回実行開始時に STATE.json を読み、`next_day` と未実行タスクから続きを実行する。
- STATE更新 → commit は各Day完了のたびに行い、中断耐性を確保する。

## 3. 継続優先（完璧不要）

- ツールの完成度は最高品質でなくてよいが、「適当すぎ」「似たり寄ったり」を防ぐ。
- 失敗時は **無限リトライ禁止**。止まって報告し、次の一手を提示する。
- 最大リトライ回数：**2回**（初回 + 2回で計3試行が上限）。

## 4. バッチサイズ

- `batch_size_default = 7`（固定・厳守）
- 1回の起動で最大7本を処理する。

## 5. Diversity Gate（多様性ゲート）

各Day repoの `meta.json` に以下を必須で含める：

| フィールド | 説明 | 例 |
|-----------|------|-----|
| `core_action` | 何をするか | convert, compare, generate, track, visualize, format |
| `twist` | 差別化1点 | 保存, エクスポート, 可視化, ショートカット, スキン, 履歴, プリセット |
| `one_sentence` | 1文説明 | 「画像をドラッグ&ドロップでWebPに変換」 |
| `keywords` | キーワード（任意） | ["image", "converter", "webp"] |

### 判定ルール

- `recent_meta`（直近20本）と比較し：
  - `core_action` が **3連続** したら → 作り直し
  - `twist` が空 / 弱い / 既視感が強い → 作り直し
  - `one_sentence` が直近と明らかに類似 → 作り直し
- 判定は簡易文字列一致 or AI判定のどちらでもよい。

## 6. Build & Smoke Gate

- `npm ci && npm run build` が成功すること。
- preview相当でトップページが開くこと。
- 主要操作が1つ以上動くこと（入力→出力、ボタン→結果など）。
- **失敗したらそのDayは完了扱いにしない**。原因と次の一手を提示して停止。

## 7. README Gate

各Day repoの README.md に以下を必須で含める：

1. 1行説明
2. 使い方（最大3箇条）
3. GitHub Pages URL
4. Day表記（DayXXX / #100日開発）

## 8. X投稿ルール

### 固定タグ（短縮なし・毎回同じ）

```
#個人開発 #100日開発
```

### 投稿テンプレート

**標準:**
```
Day{XX}｜{ツール名}
{一言で何ができる}
{URL}
#個人開発 #100日開発
```

**文字数超過時の縮小ステップ（この順に実行）:**

1. 絵文字・空行・冗長表現（です/ます、〜できます）を削除
2. 説明文を名詞句に圧縮（24全角目安）
3. ツール名を短縮（16全角目安）
4. 圧縮版テンプレに切替
5. 最小版（最終手段）

### Buffer投入

- 「Add to Queue」相当で7本投入。
- 失敗時：標準 → 圧縮 → 最小の順で最大2回まで再試行。
- 最小でも失敗なら **停止して報告**（勝手に投稿しない）。
- 外部連携未設定/失敗なら、予約投入は停止して明確に報告。

## 9. repo命名規則

| 種別 | 命名パターン | 例 |
|------|-------------|-----|
| Control repo | `ai-dev-exp-control` | — |
| Template repo | `ai-dev-exp-template` | — |
| Day repo | `ai-dev-day-XXX` | `ai-dev-day-001`, `ai-dev-day-042` |

- Day番号は必ず **3桁ゼロ埋め**。

## 10. エラー対応

- 無限リトライ禁止。
- エラー発生時は以下を出力して停止：
  1. どこまで完了したか
  2. 原因
  3. 次の一手（具体的なアクション）
